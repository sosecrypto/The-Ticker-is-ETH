[베이스, 캐논 테스트 결과 공유]

캐논 (Cannon)

- 캐논은 옵티미즘 팀에서 만든 OP Stack 최초의 결함 증명 챌린지 시스템으로, Go로 구현되어 있음

- 캐논은 매우 단순하게 구현해놔서 여러 작업을 동시에 하는 게 사실상 불가능함

- 단순하다는 한계로 캐논의 챌린지 과정이 매우 느리므로, 챌린지 참여자들에게 각각 주어지는 3일의 시간이 부족할 수 있음

캐논 테스트

- 로컬 데브넷에서 반복해서 챌린지 시스템을 테스트할 수 있는 오픈 소스 도구 개발

- 해당 도구를 통해서 가스 타겟을 늘려가며 여러 테스트를 진행함

캐논 테스트 결과

- p256Verify와 ECAdd프리컴파일이 가장 리소스 제한에 영향을 미친다는 것을 알게 됨

- 캐논의 리소스 사용량은 검증 중인 단일 L2 블록의 내용 뿐만 아니라, 동일한 배치에 포함된 다른 L2 블록의 내용에 의해서도 영향을 받음

캐논의 목표

- 현재 버전의 캐논으로도 지금보다 3배 스케일업 할 수 있지만, 목표인 1Gigagas / sec에는 훨씬 미치지 못함

- 현재 OP Stack 기여자들은 멀티쓰레드 캐논과 64비트 캐논 업그레이드에 집중하고 있음

베이스는 오늘도 열심히 OP Stack을 위해 노력하고 있네요.

옵티미즘이 수평적 확장에 집중한다면, 베이스는 수직적 확장에 집중하는 모습입니다.
