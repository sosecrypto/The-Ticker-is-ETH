비탈릭 : DVT를 이더리움에 내장하자

요약 및 결론

- 이더리움 외부 모듈인 DVT는 밸리데이터 운영의 안정성을 크게 높여주고, 경우에 따라 추가 수익까지 제공하면서 대형 밸리데이터 오퍼레이션을 중심으로 사실상의 이더리움 인프라의 기본 스택이 되어가고 있다.

- 하지만 현재의 DVT 구조는 밸리데이터 키 서명을 외부 모듈에 의존해야 한다는 점, 별도의 합의 레이어로 인한 네트워크 지연, 그리고 BLS 선형성에 대한 의존으로 인한 “장기적인 양자 저항성 문제” 라는 한계를 안고 있다.

- 비탈릭은 이러한 문제를 해결하기 위해, DVT 자체를 이더리움 프로토콜 안으로 흡수하자는 의견과 설계 방향을 제시했다.

- 이는 최근 비탈릭이 반복적으로 밀고있는, “외부 의존성을 줄이고, 이더리움을 더 단순한 시스템으로 만들자”는 주장과도 일관된 흐름으로 보인다. (이와 관련해서는 별도의 글로 정리해볼 예정이다.)

- 이 제안이 가장 직접적인 영향을 받는 프로젝트는 SSV와 Obol일 수밖에 없다.

- 해당 제안이 실제로 이더리움 메인넷에 적용될지는 알 수 없다. 다만 정말로 로드맵에 포함될 경우, SSV와 Obol은 다른 살길을 찾아내지 않는다면 살아남기 힘들 것이다.

- SSV는 SSV 2.0을 통해 리스테이킹 영역으로 확장을 시도하고 있지만 잘 되지 않는것으로 보인다. Obol의 경우에는 현재 퍼포먼스와 운영 복잡성 문제를 먼저 해결해야 하는 과제가 남아 있어, DVT 외의 방향으로 확장하기까지는 시간이 필요해 보인다.

방식에 대한 쉬운 비유

- DVT : 원래 하나였던 키를 여러 조각으로 쪼개 두었다가, 밖에서 다시 합의해 하나의 키로 합쳐 금고를 여는 방식

- 비탈릭 제안 : 처음부터 서로 다른 키를 여러 개 만들어 두고, 그중 정해진 개수 이상이 동시에 꽂히면 금고가 열리도록 금고 자체를 바꿔버리는 방식
