🔹 EIP-8025: Optional Execution Proofs (선택적 실행 증명 제안)

1. 핵심 개념

- 비콘 노드가 실행 레이어 클라이언트 없이 블록 검증 가능

- 실행 증명을 컨센서스 레이어 P2P 네트워크로 전송

- 기존: 검증자가 EL 클라이언트 필수 → 제안: 증명만으로 검증

2. 도입 목적

- 검증자의 하드웨어·대역폭 요구사항 감소

- 가스 한도가 늘어나도 블록 검증 비용 증가하지 않음

- 선택 사항이라 프로토콜 업그레이드가 이 개선사항에 의존하지 않음

- 컨센서스 크리티컬하지 않게 안전하게 테스트 가능

3. 작동 방식

- 검증자가 zkEVM 증명 생성 모드 또는 무상태 검증 모드 선택 가능

- 증명 생성 노드: 비콘 블록 받으면 실행 페이로드 증명 생성 → 지정 서브넷에 재전파

- 무상태 검증 노드: 비콘 블록 받으면 해당 서브넷에서 유효한 증명 받았는지 확인

- 컨센서스 레이어만 변경 (실행 레이어 변경 없음)

4. 구현 상태

- Draft 단계, 세부 구현·테스트·보안 고려사항 미완성

- 하위 호환성 유지, 하드포크 불필요

실행 증명을 안전하게 테스트하기 위한 초기 제안, 구체적 구현 방법은 개발 중
