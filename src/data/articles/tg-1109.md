EIP 시리즈, Glamsterdam 가스 리프라이싱: 이더리움 가스 체계가 바뀐다

원문 X 아티클

이더리움의 다음 하드포크 Glamsterdam(2026 상반기)에서 EVM opcode별 가스 비용을 실제 자원 소비에 맞게 재조정하는 가스 리프라이싱이 논의 중이다.

1. 왜 필요한가

현재 가스 테이블은 수년간의 클라이언트 최적화와 하드웨어 변화를 반영하지 못하고 있다. 

핵심 문제는 worst-case block size인데, 저평가된 opcode를 악용하면 블록 크기를 비정상적으로 키울 수 있고, 이는 노드 처리 시간을 극단적으로 늘려 네트워크 안정성을 위협할수 있다.

주요 EIP 5가지이며 모두 검토 중이고 아직 Glamsterdam에 포함 여부는 확정되지 않았다.

• EIP-7904: opcode별 가스 비용 전면 재조정

• EIP-8011: 특정 opcode 가스 비용 세부 조정

• EIP-8032: 자원 유형별 차등 가스 비용

• EIP-8037: 다차원 가스 미터링 (상태 생성 별도 계량)

• EIP-8038: 혼잡 기반 동적 가스 조정

2. 가스 리프라이싱 핵심 제안 : EIP-8037

현재 이더리움 가스 구조는 연산/상태 생성/calldata가 단일 "가스"로 합산된다. 

EIP-8037은 이를 자원 유형별 분리 계량으로 바꾸자는 제안이다.

• 연산: CPU 사이클 기반

• 상태 생성: 디스크 I/O + 영구 저장 비용

• calldata: 네트워크 대역폭 기반

상태 생성 비용이 저평가되어 이더리움 상태 데이터가 매달 약 2%씩 증가 중이다. 분리 계량 도입 시 상태 성장을 경제적으로 제어할 수 있다.

3. 이해관계자별 체감

• 일반 사용자 : 단순 전송은 변화 미미. 복잡한 컨트랙트 이용 시 비용 변동 가능. 솔직히 체감하기 어렵다. 

• L2/롤업 : calldata 비용 재조정 + PeerDAS 결합으로 DA 비용 구조 재편 가능. 

• 밸리데이터 : ePBS로 MEV 수익 구조 변화. 상태 생성 비용이 인상될 경우, 상태 조작 기반 MEV 전략 수익성 감소 가능.
