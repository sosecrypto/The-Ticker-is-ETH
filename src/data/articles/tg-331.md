🙂 L2BEAT가 구워주는 피자는 무엇일까?

L2BEAT에서 만우절을 기념해서 피자를 구워주는 이벤트를 했습니다.

채소, 토마토, 치즈를 선택하면 자신과 어울리는 레이어 2를 추천해주는 기능이었습니다.

사실 이 피자는 레이어 2의 리스크를 다섯 개로 분류하고, 리스크에 따라 빨간색, 노란색, 초록색으로 분류한 것입니다.

1. 상태 검증(State Validation)

시스템 상태의 유효성은 어떻게 검증되는가?

-빨간색: 기능 없음

-노란색: 결함 증명 혹은 영지식 증명이 도입되어있지만, 중앙화된 주체가 악의적으로 행동할 수 있음

-초록색: 결함 증명 혹은 영지식 증명이 도입되어있고, 탈중앙화되어있음

2. 데이터 가용성(Data Availability)

상태를 재구성하는 데 필요한 데이터가 이용 가능한가?

-빨간색: 외부 DA 레이어 사용

-초록색: 이더리움 사용

3. 탈출 가능 시간(Exit Window)

원하지 않는 업그레이드가 발생했을 때, 사용자가 시스템에서 나올 수 있는 시간은 얼마나 되는가?

-빨간색: 기능 없음

-노란색: 30일 이내 업그레이드가 이루어짐

-초록색: 업그레이드에 30일 이상이 소요됨

4. 시퀀서 장애(Sequencer Failure)

시퀀서가 오프라인이 되거나 특정 사용자를 검열할 경우 어떻게 되는가?

-빨간색: 기능 없음

-노란색: 사용자가 이더리움으로 인출은 가능하지만, 검열 가능성 존재

-초록색: 사용자가 검열 없이 메인넷으로 인출 가능

5. 프로포저 장애(Proposer Failure)

프로포저가 오프라인일 경우 어떻게 되는가?

-빨간색: 출금 불가

-노란색: 특정 프로포저들만 상태 제출 가능

-초록색: 누구나 직접 상태 제출 가능

레이어 2를 피자에 비유한 재밌는 시도인데, 아무래도 이탈리아 출신들이 만든 단체라 이런 기능을 개발한 것 같습니다.
